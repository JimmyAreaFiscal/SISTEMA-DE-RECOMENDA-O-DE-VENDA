# Sistema de Recomendação de Venda

Projeto desenvolvido para a disciplina **Projeto Aplicado III** do curso de **Ciência de Dados da Universidade Presbiteriana Mackenzie**.

O objetivo do projeto é desenvolver um **sistema de recomendação baseado em aprendizado de máquina** capaz de sugerir o **tipo de pagamento mais adequado** em um sistema de vendas de produtos comercializados em um marketplace.

# Visão Geral

O crescimento do comércio eletrônico e das plataformas digitais tem aumentado a necessidade de soluções baseadas em dados capazes de apoiar decisões estratégicas.

Nesse contexto, **Sistemas de Recomendação** têm sido amplamente utilizados para sugerir produtos, serviços ou conteúdos personalizados aos usuários. Este projeto explora uma aplicação menos comum desses sistemas: **a recomendação da forma de pagamento mais adequada para uma transação de compra**.

A recomendação do tipo de pagamento pode considerar variáveis como:

- valor da compra
- categoria do produto
- perfil do cliente
- histórico de transações

Com base nesses dados, o sistema utiliza **técnicas de aprendizado de máquina supervisionado** para identificar padrões e sugerir a forma de pagamento mais adequada.

---

# Contexto do Estudo

O projeto simula uma **empresa fictícia que comercializa produtos em um marketplace digital**, operando com diferentes categorias de produtos e múltiplas formas de pagamento, como:

- Cartão de crédito
- Boleto bancário
- Transferência digital

A base de dados utilizada contém informações históricas de vendas e será utilizada para:

- análise exploratória
- preparação e tratamento dos dados
- treinamento de modelos de machine learning
- avaliação de desempenho do modelo

O estudo busca demonstrar como **técnicas de ciência de dados podem gerar insights estratégicos em ambientes de comércio eletrônico**.

# Objetivos

## Objetivo Geral

Desenvolver um **sistema de recomendação baseado em aprendizado de máquina** capaz de sugerir o tipo de pagamento mais adequado em um sistema de vendas de produtos comercializados em marketplace.

## Objetivos Específicos

- Desenvolver modelo capaz de gerar recomendação de tipos de pagamentos para clientes

- Desenvolver e manter estrutura de dados (banco de dados) em apoio ao software e ao modelo (MLOps)

# Etapas do Projeto

O projeto segue etapas típicas de três tipos diferentes de projetos:

A) Fluxo de **Ciência de Dados**

1. Coleta de dados  
2. Pré-processamento dos dados  
3. Análise exploratória (EDA)  
4. Engenharia de features  
5. Treinamento do modelo  
6. Avaliação de desempenho  
7. Análise dos resultados
8. Deployment

B) Fluxo de **Engenharia de Software**

1. Análise e elicitação de Requisitos
2. Definição de Backlog
3. Definição de Arquitetura de Software e de Infraestrutura
4. Implementação dos códigos
5. Testes
6. Validação
7. Deployment

C) Fluxo de **Engenharia de Dados**

1. Definição e extração das fontes de dados
2. Preparação do banco de dados (infraestrutura)
3. Montagem de pipeline ETL
